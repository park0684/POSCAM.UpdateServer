# B00 - Starter 분석

## 1. 작업 목적
정책, API, DB, 배포 문서의 충돌과 누락을 확인한다. 프로젝트를 생성하지 않는다.

## 2. 반드시 먼저 읽을 문서
- `AGENTS.md`
- `README.md`
- `docs`의 관련 정책
- `database/schema.sql`
- `codex/WORK_STATUS.md`
- 상위 `DECISIONS.md`

문서와 현재 코드가 충돌하면 임의 구현하지 말고 `Blocked`로 보고한다.

## 3. 작업 전 현재 상태 확인
- 정책과 API 일치
- schema.sql과 V001 일치
- 오류 코드
- AuthServer 계약
- Storage·Nginx 경로
- 단계 의존성

## 4. 수정 허용 범위
- 코드·프로젝트 생성 금지
- 분석 보고와 B00 상태

목록 밖 파일이 반드시 필요하면 이유와 경로를 먼저 보고한다.

## 5. 금지 사항
- 정책 변경
- DB 컬럼 변경
- 선행 구현

## 6. 구현 요구사항
- 충돌·누락·모호성을 파일 경로와 함께 보고
- B01 진행 가능 여부
- 필요 SDK와 도구 기록

## 7. 오류 처리 및 안정성
- 모호성을 임의 기본값으로 처리하지 않음

## 8. 빌드 및 테스트
- 파일·문서 정합성 확인

실패 상태에서 다음 단계로 이동하지 않는다.

## 9. 완료 조건
- 코드 변경 없음
- 충돌 없음 또는 Blocked
- B01 선행조건 기록

## 10. 완료 보고 형식
`codex/COMPLETION_REPORT_TEMPLATE.md` 형식을 사용하고 `codex/WORK_STATUS.md`의 해당 단계만 갱신한다.
